# Daily Progress Summary

生成每日开发进度总结报告，整理当天完成的工作、遇到的问题和次日计划。

## 使用场景

- 用户请求生成开发进度报告
- 用户说"总结今日进度"、"日报"、"daily report"
- 用户说"开发进度"、"整理进度"

## 输入

- 当前项目的 git commit 历史
- todo.md 和 work_done.md 文件（如存在）
- 当天完成的任务记录

## 输出

Markdown 格式的日报，包含：

1. **今日完成**：列出当天完成的主要工作
2. **进行中**：正在开发的功能或任务
3. **问题与阻塞**：遇到的问题或 blockers
4. **明日计划**：次日计划开展的工作

## 实现步骤

1. 读取 git log 获取今天的提交记录
2. 检查 todo.md 和 work_done.md
3. 整理并生成报告
