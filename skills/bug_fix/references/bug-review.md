# Bug分析 Review 清单 & 独立修复计划 Review 规范

> 本文档定义 Phase 1.5（Bug分析 Review）和 Phase 2.5（独立修复计划 Review）的 Review 机制和检查清单。
>
> **核心原则**：Review Agent 必须以"接手者"视角进行审查，不持有前序 Phase 的上下文，独立、严格、不带偏好。

---

## 独立 Review 通用机制

### 执行方式

**Review Agent 启动时：**
- 不自动加载前序 Phase 的对话上下文
- 只读取必要的文档：`.dev_doc/<bug>/bug-analysis.md`、`.dev_doc/<bug>/fix-plan.md`、相关模块代码
- 以"刚接手这个Bug的新开发者"视角进行审查

**Review 结论：**
| 结论 | 含义 | 后续动作 |
|------|------|----------|
| ✅ 通过 | 无严重问题或已知可接受的风险 | 进入下一 Phase |
| ❌ 不通过 | 发现需要修正的问题 | 返回对应 Phase 调整后，重新执行独立 Review |

**Review 失败处理流程：**
```
Review Agent 报告问题
    ↓
返回对应 Phase（Phase 1 → Phase 1，Phase 2 → Phase 2）
    ↓
AI 根据 Review 意见调整Bug分析/修复计划
    ↓
重新执行独立 Review（必须再次通过）
```

## Phase 1.5: Bug分析 Review 清单

**触发时机：** Bug分析完成，测试复现成功后，提交用户确认前

**Review Agent 任务：** 审查Bug分析是否准确、测试复现是否有效

### Bug分析 Review 检查清单

```markdown
# Bug分析 Review 报告

## 1. Bug信息核对
- [ ] 报错信息是否与用户报告一致？
- [ ] 场景描述是否准确反映了触发条件？
- [ ] 相关模块是否确实与Bug相关？

## 2. 根因分析审查
- [ ] 根因分析是否准确？
- [ ] 是否找到了真正的代码问题，而不是表象？
- [ ] 调用链追踪是否完整？

## 3. 测试复现审查
- [ ] 测试用例是否能够稳定复现Bug？
- [ ] 测试用例是否独立，不依赖其他测试？
- [ ] 报错信息是否与用户报告一致？
- [ ] 复现步骤是否清晰可执行？

## 4. 代码定位审查
- [ ] 问题代码定位是否准确？
- [ ] 是否识别了所有受影响的代码路径？
- [ ] 是否遗漏了其他可能的触发条件？

## 5. 潜在风险识别
- [ ] 修复时可能引入的新问题？
- [ ] 对其他模块的影响？
- [ ] 边界条件是否考虑完整？

## 6. 优化建议
| 问题 | 优化方案 | 优先级 |
|------|----------|--------|
| [问题1] | [优化1] | 高/中/低 |

## 7. Review 结论
- [ ] 通过 / 不通过
- [ ] 如不通过，列出必须修正的问题
```

### 自审流程

```
1. 阅读Bug分析文档和相关模块代码
2. 验证测试复现是否有效
3. 填写Review清单
4. 如发现问题 → 返回Bug分析进行调整
5. Review通过 → 进入修复计划阶段
```

## Phase 2.5: 独立修复计划 Review

**触发时机：** 修复计划完成，用户确认前

**Review Agent 任务：** 以"接手者"视角审查修复计划是否可行、是否完整

**文档输入：**
- `.dev_doc/<bug>/bug-analysis.md`
- `.dev_doc/<bug>/fix-plan.md`

### 修复计划 Review 检查清单

```markdown
# 修复计划 Review 报告

## 1. Bug覆盖检查
- [ ] 是否覆盖了Bug分析中识别的所有问题点？
- [ ] 修复方案是否针对根因，而不是表象？
- [ ] 是否有遗漏的问题点？

## 2. 修复方案审查
- [ ] 修复方案是否可行？
- [ ] 是否引入了新的问题？
- [ ] 是否符合现有代码风格和架构？
- [ ] 是否有更简单的修复方案？

## 3. 影响分析审查
- [ ] 是否识别了修复对其他模块的影响？
- [ ] 是否有破坏性变更？
- [ ] 是否考虑了向后兼容性？

## 4. 测试计划审查
- [ ] 是否有针对每个修复点的测试用例？
- [ ] 是否规划了回归测试？
- [ ] 测试用例是否可执行？

## 5. 风险评估
| 风险点 | 可能性 | 影响 | 应对 |
|--------|--------|------|------|
| [具体风险] | 高/中/低 | [影响] | [方案] |

## 6. 边界条件检查
- [ ] 边界条件是否考虑完整？
- [ ] 异常场景是否有处理？
- [ ] 空值、极大值、极小值是否有考虑？

## 7. Review 结论
- [ ] 通过 / 不通过
- [ ] 如不通过，列出必须修正的问题
```
